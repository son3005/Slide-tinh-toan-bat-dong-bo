Phần 02 · concurrent.futures

1. Tổng quan module concurrent.futures

Sự ra đời & Mục đích

Ra mắt từ phiên bản **Python 3.2**, cung cấp một giao diện cấp cao (high-level interface) đồng nhất để lập trình viên thực thi các hàm hoặc lệnh gọi một cách bất đồng bộ dưới nền.

Cơ cấu phân cấp lớp

Module được xây dựng quanh lớp trừu tượng base là Executor, chia thành hai lớp con cụ thể tương ứng với hai hạ tầng thực thi khác nhau:

  • ThreadPoolExecutor: Quản lý luồng (Dựa trên module threading).
  • ProcessPoolExecutor: Quản lý tiến trình độc lập (Dựa trên module multiprocessing).

Ưu điểm nổi bật

  • Tự động quản lý (Automated Management): Tự động quản lý vòng đời worker (khởi tạo, phân chia công việc, tắt worker), loại bỏ gánh nặng tự viết code quản lý luồng/tiến trình thủ công rất dễ lỗi.
  • Đơn giản hóa (Simplification): Giao diện API đồng nhất (cùng dùng submit, map, shutdown) giúp chuyển đổi giữa đa luồng và đa tiến trình cực kỳ dễ dàng chỉ bằng cách đổi tên lớp.
  • Tối ưu đa lõi (Multi-core Optimization): Khai thác triệt để sức mạnh phần cứng CPU đa lõi cho tác vụ tính toán nặng thông qua Process Pool.
