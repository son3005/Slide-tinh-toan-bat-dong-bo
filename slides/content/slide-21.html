Phần 02 · concurrent.futures

Raw Thread/Process vs Worker Pooling

1. Raw Thread / Process (Thủ công)

import threading
threads = []
for url in urls:
    t = threading.Thread(target=tai_web, args=(url,))
    threads.append(t)
    t.start()
for t in threads:
    t.join()
  • Quản lý thủ công: Lập trình viên phải tự viết code tạo, kích hoạt (`start()`) và chờ thu hồi (`join()`) từng luồng/tiến trình con.
  • Thread Explosion (Bùng nổ luồng): Nếu có 1,000 URLs, code sẽ tạo ra 1,000 luồng chạy song song, làm nghẽn CPU, cạn kiệt bộ nhớ RAM và gây sập hệ điều hành.
  • Không tái sử dụng: Luồng bị hủy hoàn toàn ngay khi chạy xong việc, tạo chi phí khởi động rất lớn cho tác vụ tiếp theo.

2. Worker Pooling (Hồ chứa Worker)

from concurrent.futures import ThreadPoolExecutor

with ThreadPoolExecutor(max_workers=4) as executor:
    executor.map(tai_web, urls)
  • Tự động quản lý: Khối lệnh with (context manager) tự động quản lý việc khởi tạo và thu hồi tài nguyên, tự gọi shutdown() an toàn.
  • Khống chế tài nguyên: Chỉ duy trì tối đa max_workers=4 hoạt động. Nếu có 1,000 URLs, chúng sẽ nằm chờ trong **Task Queue** và được xử lý cuốn chiếu bởi 4 worker.
  • Tái sử dụng (Reuse): Worker làm xong việc lập tức chuyển sang chế độ rảnh (Idle) chờ nhận việc tiếp theo, triệt tiêu độ trễ cấp phát.
