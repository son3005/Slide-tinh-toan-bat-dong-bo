Phần 02 · concurrent.futures

Kết quả chạy thử nghiệm đo thời gian

[1. Tác vụ I/O-bound (3 URLs)]

-> [Tuần tự] tốn: 3.00 giây
-> [ThreadPool] tốn: 1.00 giây

Do 3 task chạy đan xen song song ngầm trên 3 thread, tổng thời gian tốn chỉ bằng thời gian của task lâu nhất (1 giây so với 3 giây tuần tự).

[2. Tác vụ CPU-bound (4 inputs)]

-> [Tuần tự] tốn: 2.22 giây
-> [ThreadPool] tốn: 2.62 giây (Tệ hơn!)
-> [ProcessPool] tốn: 0.70 giây (Tối ưu!)

* **ThreadPool** tốn nhiều thời gian hơn cả tuần tự do khóa **GIL** của Python ngăn cản đa luồng chạy CPU thực tế, cộng thêm chi phí chuyển ngữ cảnh (context switch). * **ProcessPool** tăng tốc rõ rệt nhờ chia việc chạy trên 4 nhân CPU thực tế, lách hoàn toàn rào cản khóa GIL.
