4. Phân tích trạng thái CPU & Mô phỏng Concurrency

Trạng thái Bận (Busy)

CPU đang thực hiện các tính toán logic, chạy mã lệnh (thuật toán, render đồ họa, xử lý dữ liệu).

Trạng thái Nhàn rỗi (Idle)

CPU tạm dừng tính toán để chờ đợi các thao tác đọc/ghi từ đĩa hoặc nhận dữ liệu phản hồi từ card mạng (I/O Wait). Gây lãng phí rất lớn tài nguyên phần cứng.

Tối ưu hóa Đồng thời (Concurrency)

Khi Task A chờ I/O (Idle), CPU lập tức chuyển sang chạy tính toán nhỏ của Task B, giữ CPU luôn ở trạng thái Busy, tối đa hóa hiệu suất phần cứng.
