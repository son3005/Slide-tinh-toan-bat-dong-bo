Chiếc Cầu Nối: run_in_executor()

Cách Event Loop (Não bộ) ủy quyền công việc cho Worker Pool (Cơ bắp)

✅ Đoạn code Chuẩn mực
import asyncio, requests
from concurrent.futures import ThreadPoolExecutor

# Tạo "Cơ bắp" (Nhà máy xử lý I/O Blocking)
pool = ThreadPoolExecutor(max_workers=10)

async def download_image():
    loop = asyncio.get_running_loop()
    
    # THẦN CHÚ: Ném việc cho Pool, Event Loop đi làm việc khác!
    response = await loop.run_in_executor(pool, requests.get, "https://...")
    
    return response.content

Cơ chế Ủy Quyền

  • Event Loop rảnh tay: Khi gọi run_in_executor(), não bộ (Event Loop) sẽ gói hàm requests.get lại và ném sang nhà máy Worker Pool. Ngay lập tức nó quay về phục vụ các kết nối khác.
  • Worker làm việc bẩn: Các Worker (Thread) trong Pool sẽ gánh chịu sự "treo" của hàm tải file.
  • Từ khóa await: Giúp Event Loop "treo" hàm download này lại, nhưng không treo toàn bộ hệ thống. Khi nào Worker tải xong, nó sẽ báo cho Event Loop để chạy tiếp.
