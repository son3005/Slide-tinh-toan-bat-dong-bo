4.2 Chiếc cầu nối: run_in_executor()

Cơ chế Ủy Quyền

Cách duy nhất để giải quyết vấn đề hàm Blocking là tống cổ nó ra khỏi Event Loop và giao cho một "cơ bắp" (Thread/Process) khác xử lý.

  • Event Loop rảnh tay: Khi gọi run_in_executor(), não bộ (Event Loop) sẽ gói hàm requests.get lại và ném sang nhà máy Worker Pool. Ngay lập tức nó quay về phục vụ các kết nối khác.
  • Worker làm việc bẩn: Các Worker (Thread) trong Pool sẽ gánh chịu sự "treo" của hàm tải file.
  • Từ khóa await: Giúp Event Loop "treo" Coroutine hiện tại, nhưng không treo toàn hệ thống. Khi nào Worker tải xong, nó sẽ báo cho Event Loop để chạy tiếp.

Đoạn Code Chuẩn mực

Kết hợp hoàn hảo giữa Asyncio (não) và ThreadPool (cơ bắp).

import asyncio, requests
from concurrent.futures import ThreadPoolExecutor

# Tạo "Cơ bắp" (Nhà máy xử lý I/O Blocking)
pool = ThreadPoolExecutor(max_workers=10)

async def download_image():
    loop = asyncio.get_running_loop()
    
    # THẦN CHÚ: Ném việc cho Pool, Event Loop rảnh tay!
    response = await loop.run_in_executor(
        pool, 
        requests.get, 
        "https://..."
    )
    
    return response.content
