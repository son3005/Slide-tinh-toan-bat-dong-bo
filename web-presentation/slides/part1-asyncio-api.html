async def fetch_data(id):

Khu Vực Phân Tích

Chọn một từ khóa bên trái để xem cách hoạt động của nó ở cấp độ lõi của Event Loop.

async def (Coroutine)

Hàm bình thường (def) khi được gọi sẽ chạy ngay lập tức. Nhưng async def thì khác!

Khi bạn gọi fetch_data(1), code chưa hề chạy. Nó chỉ trả về một đối tượng Coroutine (giống như một hộp chứa mã lệnh chờ được thực thi).

fetch_data(1)
➔
Coroutine Object
(Chưa chạy, đang đợi Event Loop)

await (Nhường quyền)

Đây là linh hồn của Asyncio. Khi gặp await, tác vụ hiện tại sẽ bị treo (đứng im).

Nhưng thay vì làm treo cả chương trình, await sẽ trả quyền điều khiển (Yield) lại cho Event Loop, để Event Loop chạy đi làm việc khác!

Task 1
Event Loop
➔ Yield (Nhường quyền)

asyncio.create_task()

Lệnh này gói Coroutine thành một Task và ném thẳng vào hàng đợi của Event Loop.

Nó sẽ lập tức chạy ngầm dưới nền mà không cần bạn phải await ngay lúc đó (Non-blocking). Cực kỳ hữu dụng khi muốn kích hoạt task chạy song song.

Event Loop Queue
Task Cũ

asyncio.gather()

Gom nhóm nhiều Coroutines lại và yêu cầu Event Loop chạy TẤT CẢ chúng cùng một lúc.

Nó sẽ chờ (block) cho đến khi toàn bộ task trong nhóm chạy xong, sau đó gộp kết quả lại thành một mảng (List) theo đúng thứ tự truyền vào.

fetch(2)
fetch(3)
fetch(4)
