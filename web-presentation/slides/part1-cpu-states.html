1.1 Trạng thái CPU (Busy vs Idle)

CPU vật lý luôn hoạt động ở 2 trạng thái cơ bản. Mục tiêu tối thượng của lập trình đồng thời là giữ cho CPU luôn ở trạng thái Busy.

Trạng thái Busy (Bận rộn)

CPU đang tích cực thực thi các tập lệnh tính toán phức tạp (Toán học, Mã hóa, Xử lý ảnh AI).

Trạng thái Idle (Nhàn rỗi)

CPU không có lệnh để chạy, phải đứng chờ các thiết bị ngoại vi chậm chạp (Mạng lưới internet, Ổ cứng từ tính).

Nút thắt cổ chai: Tốc độ mạng/ổ cứng chậm hơn tốc độ CPU hàng triệu lần!

Thời gian biểu thực thi

Cách 1: Lập trình Tuần tự (Lãng phí)

CPU Tính toán
IDLE (Chờ Mạng tải File)
Lưu File
Tổng thời gian: 10s

Cách 2: Lập trình Bất đồng bộ (Tối ưu)

Tính Task 1
Tính Task 2
Tính Task 3
Chờ Mạng chung
Tổng thời gian: 5s

Tranh thủ lúc chờ I/O của Task 1, ta nhét Task 2 và 3 vào chạy trên CPU!
